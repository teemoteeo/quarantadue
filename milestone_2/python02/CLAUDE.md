## Project Overview

42 School "Garden Guardian" project (python02) — Python exception handling module. Exercises progressively teach try/except, custom exceptions, finally blocks, raise, and combining all techniques into a garden management system.

## Requirements

- Python 3.10+
- Code must pass **flake8** linter
- All functions and methods must include **type hints**
- Programs must never crash (graceful error handling is the point)
- Each exercise is a standalone `.py` file in its own `exN/` directory

## Running and Linting

```bash
# Run any exercise
python3 ex0/ft_first_exception.py

# Lint
flake8 ex0/ft_first_exception.py
```

## Exercise Structure

| Ex | Directory | File | Authorized Functions | Topic |
|----|-----------|------|---------------------|-------|
| 0 | ex0/ | ft_first_exception.py | int(), print() | Basic try/except with ValueError |
| 1 | ex1/ | ft_different_errors.py | print(), open(), close(), int() | Multiple exception types (ValueError, ZeroDivisionError, FileNotFoundError, KeyError) |
| 2 | ex2/ | ft_custom_errors.py | print(), int(), input() | Custom exception classes (GardenError -> PlantError, WaterError) |
| 3 | ex3/ | ft_finally_block.py | print(), int() | finally blocks for resource cleanup |
| 4 | ex4/ | ft_raise_errors.py | print(), int() | Raising exceptions with raise keyword |
| 5 | ex5/ | ft_garden_management.py | print(), int() | GardenManager class combining all techniques |

## Key Patterns

- Each exercise file has a shebang `#!/usr/bin/env python3` and a `if __name__ == "__main__":` guard calling a test function
- **Authorized functions** are strictly limited per exercise — only use what's listed
- Custom exceptions in ex2 use inheritance: `GardenError(Exception)` as base, `PlantError(GardenError)` and `WaterError(GardenError)` as children
- Ex5 is the capstone: integrates custom exceptions, try/except/finally, and raise into a `GardenManager` class
